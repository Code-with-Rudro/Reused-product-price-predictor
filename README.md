## 📖 About the Project

The **Reusable Product Price Predictor** is an AI/ML-powered web application that estimates the **fair resale price** of used products based on their specifications and usage history.

Currently supports **Cars** — with a reusable architecture designed to extend to:

| Category | Status |
|---|---|
| 🚗 Cars | ✅ Live |
| 📱 Mobiles | 🔜 Coming soon |
| 🏍️ Bikes | 🔜 Coming soon |
| 🪑 Furniture | 🔜 Coming soon |

Built with a **FastAPI** backend, a **Streamlit** frontend, and a **Random Forest Regressor** ML model trained on real CarDekho resale data.

---

## 🚨 Problem Statement

In online resale platforms like **OLX**, **Facebook Marketplace**, and similar sites:

- 📊 **65–75%** of users face confusion about fair pricing
- 📉 **50–60%** of products are listed at inflated or inconsistent prices
- ✅ Only **25–35%** of listings are at a genuinely fair market value

This leads to:
- Buyers overpaying or losing trust in the platform
- Sellers struggling to price competitively or losing money
- Delayed, inefficient transactions

**Our solution:** A machine learning model that takes product details as input and returns a data-backed predicted price — giving sellers a fair anchor and buyers a trust signal.

## ✨ Features

## 🚀 Getting Started

### Prerequisites

Make sure you have the following installed:

- **Python 3.10+** → [Download](https://python.org/downloads)
- **pip** (comes with Python)
- **Git** → [Download](https://git-scm.com)

---

### Installation

**1. Clone the repository**

```bash
git clone https://github.com/your-username/car-price-api.git
cd car-price-api
```

**2. Create and activate a virtual environment**

```bash
# Create virtual environment
python -m venv venv

# Activate — Windows
venv\Scripts\activate

# Activate — macOS / Linux
source venv/bin/activate
```

**3. Install dependencies**

```bash
pip install -r requirements.txt
```

**4. Train the ML model** *(only needed once — generates the `.pkl` files)*

```bash
python train.py
```

After this step you should see:
```
✅ random_forest_model.pkl
✅ feature_columns.pkl
```

---

### Running the Backend (FastAPI)

```bash
uvicorn main:app --reload
```

The API will be live at:

```
http://127.0.0.1:8000
```

Visit the interactive API docs at:

```
http://127.0.0.1:8000/docs
```

---

### Running the Frontend (Streamlit)

Open a **new terminal** (keep the FastAPI server running in the first one), then:

```bash
streamlit run streamlit_app.py
```

The app will open automatically in your browser at:

```
http://localhost:8501
```

## 📡 API Reference

### `POST /predict`

Predicts the resale price of a used car.

**Request Body**

```json
{
  "Car_Name": "maruti swift",
  "Year": 2019,
  "Present_Price": 7.5,
  "Kms_Driven": 35000,
  "Seller_Type": "Dealer",
  "Transmission": "Manual",
  "Owner": 0
}
```

**Field Descriptions**

| Field | Type | Description |
|---|---|---|
| `Car_Name` | `string` | Name/model of the car |
| `Year` | `integer` | Year of purchase (e.g. `2019`) |
| `Present_Price` | `float` | Current ex-showroom price in lakhs |
| `Kms_Driven` | `integer` | Total kilometres driven |
| `Seller_Type` | `string` | `"Dealer"` or `"Individual"` |
| `Transmission` | `string` | `"Manual"` or `"Automatic"` |
| `Owner` | `integer` | Number of previous owners (`0`, `1`, `2`, `3`) |

**Success Response — `200 OK`**

```json
{
  "prediction_price": 5.47
}
```

> 💡 `prediction_price` is in **lakhs (₹)**

**Error Responses**

| Status Code | Meaning | Fix |
|---|---|---|
| `405 Method Not Allowed` | Sent a GET request instead of POST | Change request method to `POST` |
| `422 Unprocessable Entity` | Missing or invalid input fields | Check all fields match the schema above |

## 🤖 ML Pipeline

```
Raw Data (CSV)
     ↓
Data Preprocessing
  - Handle missing values
  - Drop irrelevant columns
     ↓
Feature Encoding
  - Label encode categorical fields
  - (Seller_Type, Transmission)
     ↓
Train / Test Split
  - 80% Training  |  20% Testing
     ↓
Model Training
  - Random Forest Regressor
  - Handles non-linear relationships
  - Robust to outliers
     ↓
Model Evaluation
  - R² Score, MAE, RMSE
     ↓
Save Model
  - random_forest_model.pkl
  - feature_columns.pkl
     ↓
FastAPI Integration
  - Load .pkl on startup
  - Preprocess input → Predict → Return price
```

**Why Random Forest?**
- Handles non-linear relationships between features and price
- Robust against overfitting on structured/tabular data
- High accuracy without extensive hyperparameter tuning
- Works well with mixed feature types (numeric + categorical)

| Column | Description |

## ⚙️ How It Works

```
User fills form on Streamlit
          ↓
POST request → FastAPI /predict
          ↓
Pydantic schema validates input
          ↓
model.py loads random_forest_model.pkl
          ↓
Input is preprocessed (encoded & aligned to feature_columns)
          ↓
Model predicts resale price
          ↓
{ "prediction_price": X.XX } returned
          ↓
Streamlit displays the estimated price to the user
```

## 🐛 Known Issues & Fixes

| # | Error | Cause | Fix |
|---|---|---|---|
| 1 | `pip upgrade failed` | Outdated pip version | Run `python -m pip install --upgrade pip` |
| 2 | `venv not activating` | Wrong directory or OS command | Use `venv\Scripts\activate` (Windows) or `source venv/bin/activate` (Mac/Linux) |
| 3 | `405 Method Not Allowed` | Sending GET to a POST endpoint | Use POST method in Postman or hit `/docs` Swagger UI |
| 4 | `422 Unprocessable Entity` | Wrong field names or missing data | Double-check request body matches schema exactly |
| 5 | `random_forest_model.pkl not found` | Model not trained yet | Run `python train.py` first |
| 6 | `feature_columns.pkl not found` | Same as above | Run `python train.py` first |
